# 简历列表

## 目标

简历列表展示用户所有简历版本，并作为进入简历详情、AI 对话、修改日志和完整编辑器的入口。

## 简历来源

来源字段必须明确：

- `manual_upload`：用户手动上传或 onboarding 解析生成。
- `target_job`：针对某个 JD 由 AI 生成。

处理状态必须明确：

- `uploaded`：文件已上传，尚未解析。
- `parsing_pending`：等待 Dify 简历解析 Workflow。
- `parsed`：解析完成，等待用户确认或已生成 base resume。
- `parse_failed`：解析失败，可重试或手动填写。
- `generation_pending`：等待 Dify 定制简历 Workflow。
- `generated`：定制简历已生成。
- `generation_failed`：生成失败，可重试或回到匹配报告。

target job 简历必须显示：

- 关联职位。
- 关联公司。
- 生成时间。
- 匹配分数。
- 使用的 profile 版本。
- 最新修改时间。
- 最近一次 Dify workflow 状态。

## 简历详情

简历列表中打开详情时使用侧滑组件。

详情包括：

- 简历预览。
- 来源信息。
- 关联 target job。
- AI 对话。
- 修改日志。
- 导出历史。
- 最近一次 AI run 摘要。
- 进入完整编辑器按钮。

这个侧滑组件需要和完整简历编辑器复用预览、日志和对话展示能力。

## 验收标准

- 用户能区分 base resume 和 target job resume。
- target job 简历能回到对应职位。
- 侧滑详情能展示 AI 对话和修改日志。
- 进入完整编辑器后仍使用同一份简历版本。
- 用户能看到解析/生成失败状态，并能重试或降级到手动流程。
