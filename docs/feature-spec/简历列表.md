# 简历列表

## 目标

简历列表展示用户所有简历版本，并作为进入简历详情、AI 对话、修改日志和完整编辑器的入口。

## 简历来源

来源字段必须明确，完整枚举见《Supabase 持久化》（`manual_created`、`manual_upload`、`ai_generated`、`target_job`）。列表主要区分两类：

- `manual_upload`：用户手动上传或 onboarding 解析生成。
- `target_job`：针对某个 JD 由 AI 生成。

target job 简历必须显示：

- 关联职位。
- 关联公司。
- 生成时间。
- 匹配分数。
- 使用的 profile 版本。
- 最新修改时间。
- 最近一次 Dify workflow 状态。

## 简历详情

简历列表中打开详情时使用侧滑组件。

详情包括：

- 简历预览。
- 来源信息。
- 关联 target job。
- AI 对话。
- 修改日志。
- 导出历史。
- 最近一次 AI run 摘要。
- 进入完整编辑器按钮。

这个侧滑组件需要和完整简历编辑器复用预览、日志和对话展示能力。

## 验收标准

- 用户能区分 base resume 和 target job resume。
- target job 简历能回到对应职位。
- 侧滑详情能展示 AI 对话和修改日志。
- 进入完整编辑器后仍使用同一份简历版本。
- 解析/生成失败通过当前操作的错误提示处理，不在简历列表创建失败占位记录。
