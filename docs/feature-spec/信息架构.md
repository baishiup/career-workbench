# 信息架构

## 目标

主应用采用 Jobright-style 工作台布局：左侧固定侧边栏，右侧主内容区。

## 普通用户导航

侧边栏第一阶段包含：

- 工作机会：职位列表、职位详情、JD 匹配入口。
- 简历：base resume、target job resume、编辑器入口、AI 对话和修改日志。
- Profile：个人事实库和求职偏好。
- 设置：账号、数据、隐私、AI provider、Supabase 连接状态。

## 后台入口

后台管理入口不放在普通用户侧边栏里。

管理员能力：

- 导入工作机会。
- 维护职位数据。
- 查看导入状态。
- 停用或修正职位。

## 默认进入状态

登录后根据用户状态决定入口：

- 没有 profile：进入 onboarding。
- 有 profile 但没有简历：进入简历上传/创建入口。
- 有 profile 和简历：进入工作机会。

## 验收标准

- 普通用户不会看到管理员导入入口。
- 主导航能覆盖工作机会、简历、Profile、设置。
- 空状态明确引导下一步。
- 不出现营销首页式入口。
