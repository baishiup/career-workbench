# 任务 4：规则匹配分

## 目标

列表页和详情页显示代码规则计算的匹配分与等级标签，替换 mock 分数。

## 前置条件

- 任务 1（结构化 JD 数据可读）。
- 与任务 2 / 3 无依赖，可并行。

## 边界

- 纯函数计算，不持久化、不预计算、不调 AI。
- 技能归一化用静态别名词典，不做语义匹配。

## 输入

- `profiles.profile_data` 中的技能、年限、偏好。
- 任务 1 的结构化 JD 字段（必备技能、加分技能、年限要求、远程状态）。

## 实现

- 匹配函数放共享包（按 `docs/project-structure.md` 归属，倾向 packages 下可单测的纯逻辑）：必备技能命中、加分技能命中、年限、远程偏好加权 → score；分数区间映射 强匹配 / 可冲刺 / 需补证据。
- 技能归一化词典（大小写、ReactJS → React 这类别名）。
- 列表页和详情页 UI 接入。
- 无 Profile 时显示引导完善 Profile 的空状态。

## 验收

- 匹配函数有单测：命中 / 未命中 / 归一化 / 无 Profile。
- 修改 Profile 后刷新，分数即变化。
- 空状态符合 `docs/feature-spec/工作机会.md` 的要求。
