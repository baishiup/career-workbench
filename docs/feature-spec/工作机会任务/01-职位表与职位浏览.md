# 任务 1：职位表与职位浏览接 Supabase

## 目标

建 `job_descriptions` 表，职位列表页和详情页从 Supabase 读取，mock 降级为本地 fixture。

## 前置条件

无。本任务是后续所有职位任务的基础。

## 边界

- 不含导入 UI、不含匹配分、不含任何 AI 调用。
- 写入暂时只允许 service role，admin 写策略归任务 2。

## 输入

- `docs/feature-spec/工作机会.md` 的职位字段清单。
- `apps/web/src/lib/jobs/mock-data.ts` 的 `JobRecord` 类型。
- `supabase/sql/resumes_setup.sql` 的建表风格（幂等 + RLS + check 约束）。

## 实现

- `supabase/sql/jobs_setup.sql`：字段对齐 spec（来源平台、原始链接、公司、岗位、地点、远程状态、岗位类型、年限要求、必备技能、加分技能、职责、要求、薪资、发布时间、导入方式、导入状态、启用状态、`created_at` / `updated_at`）。RLS：authenticated 可读启用职位，不建写策略。
- `supabase/seed.sql` 补 3-4 条演示职位（改写自现有 mock 数据）。
- `apps/web/src/lib/jobs/api.ts`：职位读取 API；类型从 mock-data 迁出。
- 列表页和详情页换数据源；无 Supabase env 时回落 mock 并显示模式标识。

## 验收

- 登录后列表页和详情页显示 Supabase 数据。
- 停用职位不出现在列表。
- 无 env 时 mock 可用且有模式标识。
- `pnpm check` / `pnpm test` / `pnpm build` 通过。
