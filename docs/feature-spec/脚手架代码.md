# 脚手架代码

## 目标

创建可运行的 `pnpm workspace + Vite React` 前端项目骨架，为后续功能 spec 提供统一工程基础。

## 范围

本 spec 只处理工程脚手架，不实现业务闭环。

包含：

- 初始化根目录 workspace。
- 创建 `apps/web`。
- 配置 TypeScript、Tailwind CSS、基础 lint/test/build 命令。
- 建立最小 app shell。
- 准备 mock/demo 模式的目录约定。

不包含：

- 真实 AI 调用。
- Supabase schema。
- 简历编辑器。
- 职位导入。

## 目标结构

```txt
career-workbench/
  apps/
    web/
  packages/
    ai/
    resume/
    shared/
  docs/
  package.json
  pnpm-workspace.yaml
```

第一阶段可以只创建 `apps/web` 和必要 package 占位，不强行填业务代码。

## 基础命令

需要提供：

```bash
pnpm dev
pnpm check
pnpm test
pnpm build
```

如果某个命令暂时只有占位，也必须有清晰输出，不允许静默失败。

## App Shell

首页必须是工作台雏形，不做营销页。

最小结构：

- 左侧侧边栏。
- 右侧内容区。
- 默认展示一个空状态或 mock dashboard。
- 顶部或设置区显示当前运行模式：mock / local / supabase。

## 验收标准

- `pnpm install` 成功。
- `pnpm dev` 可以启动页面。
- `pnpm check`、`pnpm test`、`pnpm build` 有明确结果。
- 没有真实 API key、真实简历或私有业务数据。
- README 能说明如何启动项目。
