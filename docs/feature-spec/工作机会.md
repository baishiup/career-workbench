# 工作机会

## 目标

工作机会由管理员导入，普通用户只浏览职位并基于职位生成 target job 简历。

## MVP 导入方式

管理员可以：

- 粘贴职位链接。
- 粘贴完整 JD 文本。
- 上传职位截图。
- 手动填写公司、岗位、地点、远程状态、技能标签和 JD。

## LinkedIn 处理策略

- MVP 不依赖服务端自动抓 LinkedIn。
- 首选手动粘贴 JD 文本或截图导入。
- 如果后续做浏览器辅助采集，只在用户/管理员本机已登录的当前页面读取可见 DOM 文本。
- 服务端不保存 cookies，不自动登录，不做绕过反爬。
- 抓取失败必须能降级到截图解析或手动粘贴。

## 职位字段

- 来源平台。
- 原始链接。
- 公司。
- 岗位名称。
- 地点。
- 远程状态。
- 岗位类型。
- 年限要求。
- 必备技能。
- 加分技能。
- 职责描述。
- 任职要求。
- 薪资范围。
- 发布时间。
- 导入方式。

## 用户侧能力

- 查看职位列表（列表页暂不显示匹配度，见下）。
- 查看职位详情。
- 查看结构化 JD。
- 触发 AI 匹配分析。
- 生成 target job 简历。

## 匹配机制

匹配度和叙事都由 AI 产出。曾经的「代码规则匹配分」已删除——静态别名词典无法处理
大小写、技术写法、技能范围/层级差异，把它表达成精确百分比是过度承诺。

### 列表页

- **暂不显示匹配度**。列表是对很多职位做粗筛，按需调用 AI 报告成本会被职位数放大，跑不起。
- 后续用 embedding 向量相似度补回一个便宜、本地、可全量的列表匹配信号，
  设计见 [列表页向量匹配](./列表页向量匹配.md)。

### AI 匹配分析（详情页）

- 用户在职位详情页主动触发，由 `job_match` 大模型读取 Profile + 结构化 JD，
  产出一个匹配度 `match_score`（0–100）和 evidence、gaps、risks、aiNote 四类叙事。
- 大模型基于语义理解评估匹配度，能看懂技术写法差异和技能范围/层级，而非字面字符串比对。
- 匹配度数字显示在详情页顶部面板，叙事显示在「匹配分析」卡片；尚未分析或报告过期时显示「—」。
- 用户没有可匹配的 Profile（技能和工作经历都为空）时，不显示匹配度，展示引导完善 Profile 的空状态。
- 结果写入 `match_reports`，每用户每职位一条，重新分析时覆盖更新。表结构见《Supabase 持久化》。
- 复用与过期：报告记录生成时的 `profile_snapshot_at` 和 `job_snapshot_at`。再次查看职位时：
  - Profile 和职位都未更新：直接展示已有分析。
  - 任一已更新：仍展示旧分析，并显示"重新分析"按钮，次级描述说明 Profile 或职位已更新。
- `resume_generate` 复用最新叙事分析作为输入；没有分析或分析已过期时，引导用户先运行分析。

## Dify 工作流

职位相关 AI 能力优先通过 Dify Workflow 编排。

- `job_parse`：把管理员粘贴的 JD 文本或职位截图转成结构化职位字段（截图由 vision 模型直接解析，不做独立 OCR）。匹配分析依赖这些字段，解析质量直接决定匹配度可用性。
- `job_match`：基于 Profile 和结构化 JD，由大模型输出匹配度（match_score 0–100）和叙事分析（evidence、gaps、risks、aiNote）。
- `resume_generate`：基于最新 match report 叙事生成 target job resume 草稿。

要求：

- 普通用户触发分析和生成，但不能修改原始职位。
- Dify 输出必须写入 `match_reports`、`ResumeVersion` 或待采纳 patch，不能只停留在 Dify 运行结果里。
- `match_reports.external_run_id` 以字符串记录 Dify `workflow_run_id`，仅用于排错；完整 AI Run Trace 体系后续接入。
- Dify 失败时职位详情仍可浏览，用户可以重试分析；本地开发 fixture 只能用于验证 UI 和事件协议。

## 验收标准

- 普通用户不能上传职位。
- 管理员能创建、编辑、停用职位。
- 职位必须有导入来源。
- LinkedIn 导入失败时能降级为手动文本或截图导入。
- 列表页不显示匹配度；详情页匹配度由 AI 产出，无可匹配 Profile 时有明确空状态。
- AI 匹配分析可复用：Profile 和职位未更新时直接展示已有报告；任一更新后展示旧报告并提示重新分析。
- AI 分析失败时职位仍可浏览，用户可重试。
