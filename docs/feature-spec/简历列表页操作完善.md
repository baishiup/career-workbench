# 简历列表页操作完善 Spec

## Summary

完善 `/resumes` 简历列表页的操作列。操作列固定显示 `查看` 和 `更多`，`更多` 下拉包含：`修改名称`、`更新到 profile`、`导出`、`删除`。

## Functional Requirements

### 查看

- 点击后读取当前简历详情并打开现有简历查看/编辑弹窗。
- 读取中显示 loading，避免重复点击。

### 更多

- 使用下拉菜单展示四个动作：
  - `修改名称`
  - `更新到 profile`
  - `导出`
  - `删除`

### 修改名称

- 点击后弹出弹窗。
- 弹窗内有一个输入框，默认填入当前简历名称。
- 名称 trim 后不能为空。
- 点击保存后更新 `public.resumes.title` 和 `updated_at`。
- 保存成功后刷新当前列表行名称；如果当前查看弹窗打开的是同一份简历，也同步标题。

### 更新到 profile

- 点击后弹出二次确认弹窗。
- 用户确认后按钮进入 loading。
- 调用现有 `apply-resume-to-profile` Edge Function。
- 该函数读取 `resumes.ai_parsed_draft_json`，重新归一化为 `ProfileDraft`，覆盖写入 `profiles.profile_data`。
- 成功后提示已更新 profile。

### 导出

- 作为预留功能。
- 点击后 toast 提示：`开发中`。
- 不做实际文件生成。

### 删除

- 点击后删除当前简历。
- 删除前判断当前用户简历数量。
- 如果只剩最后 1 份简历，不允许删除，并提示：`至少保留一份简历`。
- 删除成功后从列表移除。
- 如果被删除简历正在查看，则关闭查看弹窗。

## API Changes

新增前端 API：

- `renameResume(resumeId: string, title: string): Promise<ResumeListRow>`
  - 更新 `resumes.title`
  - 同步更新 `updated_at`
  - 返回列表所需字段

- `deleteResume(resumeId: string): Promise<{ id: string }>`
  - 先统计当前用户可见简历数量
  - `count <= 1` 时抛错
  - 否则删除对应 `resumes` 记录

复用已有 API：

- `applyResumeToProfile(resumeId: string)`
  - 不新增 Edge Function
  - 不修改数据库 schema

## UI States

- 列表加载中：保持现有 loading。
- 查看详情中：对应 `查看` 显示读取状态。
- 修改名称保存中：保存按钮显示 loading，禁止重复提交。
- 更新 profile 中：确认按钮显示 loading，禁止关闭导致重复提交。
- 删除中：禁用当前行更多操作。
- 错误统一用 toast + 页面已有错误信息模式展示。

## Validation

实施后需要验证：

- `pnpm --filter @career-workbench/web check`
- `pnpm functions:check`
- `pnpm --filter @career-workbench/web build`

浏览器验证 `/resumes`：

- 操作列显示 `查看`、`更多`。
- 修改名称能真实写入 Supabase，刷新后仍保留。
- 更新到 profile 会二次确认，并真实覆盖 `profiles.profile_data`。
- 导出只提示 `开发中`。
- 多份简历时删除成功。
- 最后 1 份简历不能删除。
- 控制台无相关 error/warn。
