# 简历编辑器

## 目标

简历编辑器用于把 AI 生成的 target job 简历继续打磨成可投递版本。

第一阶段先验证本地编辑闭环：结构化简历数据、样式配置、预览联动和 mock AI patch。暂不接 Supabase、Dify、真实 PDF 导出或生产级 canvas 排版引擎。

## 核心数据结构

简历编辑器围绕两份核心状态工作：

- `ResumeDocument`：投递用简历正文，不等同于 `ProfileDraft`。它保存 section 顺序、显隐、标题和内容块，是用户最终会投递的版本。
- `ResumeStyleConfig`：当前简历版本的样式配置。它保存模板、字体、字号、颜色、页边距、行高和 section 间距。

`ResumeVersionSnapshot` 由 `ResumeDocument` 和 `ResumeStyleConfig` 组成，后续可以作为 `ResumeVersion` 的 jsonb snapshot 持久化。

AI 修改必须先进入 `ResumePatch`。只有用户采纳后，patch 才能修改 `ResumeDocument` 或 `ResumeStyleConfig`，并写入 `ResumeChangeLog`。

## 布局

两栏布局：

- 左侧：简历预览。
- 右侧：AI Rewrite / Editor / Style 三个 tab。

## 组件集合

第一阶段需要准备以下集合组件：

- Resume Form Editor：把 `ResumeDocument` 渲染成可编辑表单，支持输入框、textarea、select、radio、tag、日期文本、section 展开收起、新增、删除和排序。
- Resume Style Editor：把 `ResumeStyleConfig` 渲染成配置表单，支持模板选择、字体、字号、颜色、间距、页边距和行高调整。
- Resume Canvas Preview：根据 `ResumeDocument` 和 `ResumeStyleConfig` 渲染简历预览。第一阶段可先用 DOM page 模拟 canvas，保留后续替换真实 canvas 排版的边界。
- Resume Overlay Layer：覆盖在预览层上方，处理 hover 高亮、点击 section、显示 `Edit with AI` 等区域级交互。
- Resume AI Chat：展示聊天消息和输入框，mock 生成 `ResumePatch`，支持采纳、拒绝和继续追问。
- Resume Patch / Change Log Panel：展示 AI patch、用户采纳/拒绝、手动编辑、排序和样式调整记录。

## 左侧预览

能力：

- 展示当前简历预览。
- 高亮 AI 修改区域。
- 支持点击区域进入编辑。
- 支持点击区域 `Edit with AI`。

## AI Rewrite

能力：

- 显示本次 AI 改写总结。
- 显示匹配分数变化。
- 显示 AI 修改列表。
- 提供快捷 prompt，例如“强化最近一段经历的动作动词”。
- 提供自由对话输入。
- 展示 AI 对话历史。
- 将 AI patch 和用户采纳/拒绝写入日志。

第一阶段 AI Rewrite 使用 mock provider。mock provider 也必须输出 `ResumePatch`，保持后续 Dify provider 的接入边界一致。

Dify 调用流程：

1. 用户点击 `Edit with AI` 或发送自由对话。
2. 前端提交当前 section、目标 JD、Profile evidence、base resume 原文和用户指令。
3. Edge Function 调用 Dify Resume Rewrite Chatflow/Workflow。
4. Dify 返回结构化 patch、解释、证据引用和风险提示。
5. 编辑器展示待采纳建议，用户选择采纳、拒绝或继续追问。
6. 只有用户采纳后，patch 才能写入当前 `ResumeVersion`。

要求：

- AI 输出必须是 patch，不允许直接覆盖整份简历。
- patch 必须可定位到 section 和 bullet，便于 compare、回滚和日志展示。
- Dify 返回内容如果缺少 evidence 或违反事实边界，前端要标记为风险建议。
- Dify 失败时保留用户输入，允许继续手动编辑，不阻断编辑器。

高亮交互：

- AI 修改过的文本在预览中高亮。
- hover 高亮文本时显示 `Compare to original`。
- 点击后弹窗展示 profile/base resume 中对应原文。
- 弹窗中标出本次改写、原文来源和风险提示。

## Editor

能力：

- 表单化展示所有简历内容。
- 支持 section 展开/折叠。
- 支持 section 拖拽排序。
- 支持 bullet 拖拽排序。
- 支持新增、删除、修改 section。
- 支持新增、删除、修改 bullet。
- 编辑后实时同步预览。

第一阶段 section：

- Personal Info。
- Professional Summary。
- Core Skills。
- Work Experience。
- Project Experience。
- Education。
- Custom Section。

## Style

能力：

- 选择模板。
- 调整字体。
- 调整字号。
- 调整边距。
- 调整行高。
- 调整 section 间距。
- 控制一页适配。

MVP 只需要 1-2 个模板，但样式控制必须是真实可用的。

第一阶段模板选择可以使用静态缩略图或简化 DOM preview；重点是模板选择和自定义样式能真实影响预览。

## 验收标准

- 编辑内容能实时同步预览。
- 预览 hover 能高亮对应 section。
- 点击预览 section 能触发表单定位事件。
- 点击 `Edit with AI` 能把 section 上下文传给 AI Chat。
- AI 修改区域能对比原文来源。
- Editor 修改不会覆盖未保存的 AI 对话结果。
- Style 调整能影响预览。
- Dify 生成的建议在采纳前不会修改 `ResumeVersion`。
- 第一阶段不依赖 Supabase、Dify、数据库 migration 或真实 PDF 导出。
