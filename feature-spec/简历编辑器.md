# 简历编辑器

_状态:✅ 已实现(表单编辑器 + 预览;AI 对话 tab 为占位)_

## 目标

简历编辑器用于把 AI 生成的 target job 简历继续打磨成可投递版本。第一阶段先验证本地编辑
闭环:结构化简历数据、样式配置、预览联动和 mock AI patch。

## 边界

- 第一阶段暂不接 Dify、真实 PDF 导出或生产级 canvas 排版引擎。
- AI 修改必须先进入 `ResumePatch`,只有用户采纳后才能改 `ResumeDocument` /
  `ResumeStyleConfig` 并写入 `ResumeChangeLog`;AI 不允许直接覆盖整份简历。
- 页面尺寸固定 A4,不提供其他纸张或自定义尺寸。
- 不做模板市场、用户上传模板或高级自定义面板。

## 用户可见行为

两栏布局:左侧简历预览,右侧 AI Rewrite / Editor / Style 三个 tab。

- **左侧预览:** 展示当前简历;高亮 AI 修改区域;支持点击区域进入编辑或 `Edit with AI`。
  高亮交互:AI 改过的文本在预览中高亮,hover 显示 `Compare to original`,点击弹窗展示
  profile/base resume 中对应原文,并标出本次改写、原文来源和风险提示。
- **AI Rewrite:** 显示本次改写总结、匹配分数变化、修改列表;提供快捷 prompt(如「强化最近
  一段经历的动作动词」)和自由对话;展示对话历史;将 patch 和用户采纳/拒绝写入日志。第一
  阶段用 mock provider,但 mock 也必须输出 `ResumePatch`,保持与后续 Dify provider 的接入
  边界一致。
- **Editor:** 表单化展示所有内容;支持 section 展开/折叠、拖拽排序,bullet 拖拽排序,新增/
  删除/修改 section 与 bullet;编辑后实时同步预览。第一阶段 section:Personal Info、
  Professional Summary、Core Skills、Work Experience、Project Experience、Education、
  Custom Section。
- **Style:** 选择模板;切换后实时更新预览的布局、颜色、联系方式图标、section 标题和技能
  展示;页面固定 A4。第一版 3 个风格差异明显的内置模板:`standard-clean`(白底单栏)、
  `business-sidebar`(左侧深色栏 + 右侧正文,模板显式声明哪些模块进侧栏)、`fresh-header`
  (顶部浅色信息区 + 单栏正文)。模板布局由模板配置决定,渲染层不得硬编码「某些模块默认进
  左栏」这类隐含规则。

## 数据与状态边界

核心状态:

- `ResumeDocument`:投递用简历正文(section 顺序、显隐、标题和内容块),不等同于
  `ProfileDraft`。
- `ResumeStyleConfig`:当前版本样式配置(内置模板 ID + 展开后的颜色、字体、间距、联系方式
  样式、section 标题样式、技能样式、布局)。
- `ResumeVersionSnapshot` = `ResumeDocument` + `ResumeStyleConfig`,后续可作为 `ResumeVersion`
  的 jsonb snapshot 持久化。

持久化:编辑结果通过右上角「保存」显式写入 `public.resumes` 的 `document_json` /
`style_json`,行 `title` 跟随 `document.title` 同步;未保存修改在离开页面时 beforeunload
提示。

组件集合(第一阶段):Resume Form Editor、Resume Style Editor、Resume Canvas Preview(可先
用 DOM page 模拟 canvas,保留后续替换真实排版的边界)、Resume Overlay Layer、Resume AI
Chat、Resume Patch / Change Log Panel。

后续 Dify 调用流程:用户点 `Edit with AI` 或发对话 → 前端提交当前 section、目标 JD、Profile
evidence、base resume 原文和指令 → Edge Function 调 Dify Rewrite Chatflow/Workflow → Dify
返回结构化 patch、解释、证据引用和风险提示 → 编辑器展示待采纳建议 → 只有采纳后 patch 才写
入当前 `ResumeVersion`。patch 必须可定位到 section 和 bullet,便于 compare、回滚和日志。

## 失败与降级

- Dify 失败时保留用户输入,允许继续手动编辑,不阻断编辑器。
- Dify 返回内容缺 evidence 或违反事实边界时,前端标记为风险建议。
- 保存失败展示错误并保留 draft。

## 验收

- 编辑内容能实时同步预览;预览 hover 能高亮对应 section;点击预览 section 能触发表单定位。
- 点击 `Edit with AI` 能把 section 上下文传给 AI Chat;AI 修改区域能对比原文来源。
- Editor 修改不会覆盖未保存的 AI 对话结果;Style 模板切换能影响预览。
- Dify 生成的建议在采纳前不会修改 `ResumeVersion`。
- 点击「保存」后 `document_json` / `style_json` / `title` 持久化,刷新后修改仍在;保存失败
  保留 draft。
- 有未保存修改时按钮可用并显示「有未保存修改」,无修改时禁用。
- 第一阶段不依赖 Dify、数据库 migration 或真实 PDF 导出。
