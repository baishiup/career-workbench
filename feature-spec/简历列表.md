# 简历列表

_状态:✅ 已实现_

## 目标

简历列表展示用户所有简历版本,并作为进入简历详情、AI 对话、修改日志和完整编辑器的入口。

## 边界

- 解析/生成失败通过当前操作的错误提示处理,**不**在简历列表创建失败占位记录。
- 列表操作列的具体行为(重命名、更新到 profile、导出、删除)见
  [简历列表页操作完善](./简历列表页操作完善.md)。

## 用户可见行为

列表按来源区分两类简历,完整枚举见 [Supabase持久化](./Supabase持久化.md)
(`manual_created`、`manual_upload`、`ai_generated`、`target_job`):

- `manual_upload`:用户手动上传或 onboarding 解析生成。
- `target_job`:针对某个 JD 由 AI 生成。

target job 简历必须显示:关联职位、关联公司、生成时间、匹配分数、使用的 profile 版本、最新
修改时间、最近一次 Dify workflow 状态。

### 简历详情(侧滑)

从列表打开详情时用侧滑组件,包含:简历预览、来源信息、关联 target job、AI 对话、修改日志、
导出历史、最近一次 AI run 摘要、进入完整编辑器按钮。该侧滑组件需与完整简历编辑器复用预览、
日志和对话展示能力。

## 数据与状态边界

- 简历来源以 `source_type` 区分(见 [Supabase持久化](./Supabase持久化.md))。
- target job 简历的关联职位/公司/profile 版本等来源信息记录在 `source_context_json`。
- 进入完整编辑器后仍使用同一份简历版本。

## 失败与降级

- 解析/生成失败通过当前操作的错误提示处理,不在列表创建失败占位记录。

## 验收

- 用户能区分 base resume 和 target job resume。
- target job 简历能回到对应职位。
- 侧滑详情能展示 AI 对话和修改日志。
- 进入完整编辑器后仍使用同一份简历版本。
